## 关于Java历史的一段简介:

该语言最初由太阳计算机系统公司的James Gosling于1991年开发<br>
当时的他正在为<b>"Green Project"</b>设计一种语言，称为<b>"Oak"</b>, <b>"Green Project"</b>致力于集中多种控制家用的电子设备。<br>
"Oak"被设计为一种强大，高效的语言，并且最大程度地可移植到不同的处理器。<br>
当然最后Greem Project失败了(不然哪来的Java)

那么在90年代初，还发生了什么呢?<br>
<b>互联网的使用</b>在90年代初开始蓬勃发展。 网页要做的已经不仅仅是显示静态文本和图形。此时的网页需用动态的交互的组件, 而问题在于，一个网页一个应用往往需要对Mac, Unix, IBM-PC, Windows分别做出适配。<br>
所以此时，一个强大，紧凑的跨平台开发语言<b>Java</b>就诞生了<br>
1994年，Sun演示了如何将Java嵌入于网页中的——被称为<b>Applets</b><br>
<b>网景浏览器</b>从1995年开始支持Applets，Java从此时开始引起了广泛的关注。<br>
Sun程序员继续开发Java的代码库，并添加了许多库。<br>
他们向大家证明了，Java不仅可以用于小应用程序，而且可以用这种高级语言编写功能完善的应用

2010年初，甲骨文收购了Sun，并致力于将Java开发为更强大的开源语言<br>
甲骨文承诺将于每六个月推出一个大版本更新<br>
现在<b>Java的最新版本</b>是<b>Java 14</b>(2020年四月)<br>

## Java的工作原理

Java语言标准（语法）在<b>所有平台上都是相同的。</b><br>
一个<b>Java编译器(Compiler)</b>被设计于将代码运行在开发平台上，它会将<b>源代码文件(.java)</b>转化为<b>字节码文件(.class)</b>这个编译器会作为<b>jdk</b>的一部分，在<b>Windows中应该是javac.exe</b>字节码文件是<b>独立于平台的(Platform Independent)</b><br>
为了让代码能够跨平台运行，所以每个平台都会通过一个<b>Java虚拟机(JVM)</b>来完成对代码的运行，<b>由于是通过虚拟机来实现多平台运行，所以运行效率较低</b>(Android系统的DVM或ART虚拟机导致应用程序运行效率较低也是同样的原因)<br>
JVM<b>接收字节码文件</b>并通过<b>生成机器代码</b>来执行该代码。<br>
后来人们将JVM从浏览器中带出，从而运行独立的Java应用程序，这就是<b>JRE(Java Runtime Engine)</b>(在Windows中就是java.exe)<br>
这样就实现了Java最初的口号：<b>"一次编译，随处运行!"</b><br>

Java可以被用于开发网页端应用，也可以被用于开发独立应用。<br>
大多数的<b>IDE(集成运行环境)</b>都允许开发这两种应用<br>
